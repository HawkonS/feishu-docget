# feishu-docget (飞书文档下载工具)

## 1. 背景与技术选型

由于公司业务需要，我们急需将大量飞书文档导出为 Word 格式以进行归档和交付。起初，我调研并使用了 [feishu2md](https://github.com/Wsine/feishu2md)（一个优秀的 Go 语言开源项目），试图通过“先转 Markdown 再转 Word”的路径解决问题。

然而在实际使用中，我们遇到了两个挑战：
1.  该项目似乎已停止维护，部分新特性（如电子表格、画板）支持不全。
2.  由于我不熟悉 Go 语言，难以对其进行二次开发和定制。

于是，我尝试基于 Python 复刻这一思路 (**Phase 1**)，开发了一套 `feishu -> markdown -> docx` 的转换工具。但在处理复杂文档时发现，“Markdown 中转”方案存在天然的局限性：
*   飞书的富文本格式（合并单元格、背景色、嵌套列表、画板等）在转为 Markdown 时会大量丢失。
*   当文档正文本身包含 Markdown 语法字符时，极易导致转换引擎解析错误，修复成本极高。

为了彻底解决格式还原问题，我决定转换技术路线 (**Phase 2**)，放弃中间格式，**直接基于飞书 API 返回的 Block 结构生成 Word 对象树 (Direct Docx)**。

**本项目即是 Phase 2 的产物**。鉴于目前工作场景对 Word 格式的需求最为迫切，我们优先完善了 Word 导出功能。在时间允许的情况下，未来也会计划重新优化并支持导出为 Markdown、HTML 等更多格式。

## 2. 核心特性

*   **⚡️ 直出 Word 文档**：
    *   跳过 Markdown 中间层，完美保留表格合并、背景色、代码块高亮等富文本格式。
*   **📝 模板系统**：
    *   支持自定义 Word 模板（`.docx`），自动应用页眉、页脚、封面。
*   **🖼 支持内容清洗**：
    *   请注意，内容清洗优先级高于模板系统：
    *   **并发下载**：多线程并行下载资源，速度极快。
    *   **画板优化**：自动下载画板并智能裁剪周围的空白区域。
*   **🧹 智能格式清洗**：
    *   **图片处理**：自动将所有图片限制在 A4 纸范围内（默认最大宽度 16cm，最大高度 23cm），并**强制居中对齐**。
    *   **缩进清洗**：强制清除正文段落、图片、表格的左侧缩进与悬挂缩进，确保文档排版整齐。
    *   **表格规范**：自动将表格宽度调整为页面宽度，并清除可能导致排版错乱的手动缩进。

## 3. 快速开始

### 3.1 环境准备

*   飞书自建应用（需开通机器人及云文档权限）

### 3.2 配置应用

在项目根目录创建 `feishu-docget.properties`（首次运行会自动生成）：

```properties
# 飞书应用凭证 (必填)
feishu.app_id=cli_axxxxxxxx
feishu.app_secret=xxxxxxxxxxxxxxxxxxxx
......
# 导出设置
image.max_width=16
image.max_height=23
download.threads=4
```

### 3.3 运行

```bash
./run.sh
```
启动后访问 `http://127.0.0.1:7800/` 即可使用。

## 4. 权限说明

机器人需要以下权限才能正常工作：
*   **云文档**：`查看、评论、编辑和管理云文档`
*   **电子表格**：`查看、评论、编辑和管理电子表格`
*   **知识库**：`读取知识库` (如需下载知识库文档)

> **注意**：对于企业内部文档，请务必在文档页面的“更多 -> 添加应用”中将机器人添加为协作者。

## 5. 常见问题

**Q: 为什么生成的 Word 中图片都在中间？**
A: 本工具内置了强制清洗规则，会将所有图片、画板统一设置为**居中对齐**，并清除缩进，以保证导出的文档在 A4 纸上排版整齐。


**Q: 提示 "Permission Denied"？**
A: 请检查飞书后台权限是否开通，以及文档是否已添加机器人为协作者。
